# 📝 Vanilla JS TodoApp

Простое, но архитектурно правильное Single Page Application (SPA) для управления списком задач. Проект создан в рамках изучения фундаментальных концепций JavaScript, работы с DOM-деревом и постоянным хранением данных на стороне клиента.

> **Ссылка на живой проект:** [Запустить Todo] *(https://invisiballos-alt.github.io/ToDo_List/)*

## Структура проекта, тех. решения

В процессе разработки этого проекта я сфокусировался на применении практик написания чистого JS:

- **Архитектура Data-Driven (Управление состоянием):** Интерфейс приложения является отражением состояния (массива объектов `todos`). Изменение данных и рендеринг производятся раздельно. Функция `renderTodos()` отвечает только за синхронизацию массива с DOM-деревом.
- **Персистентность данных (Web Storage API):** Реализовано сохранение данных с помощью `localStorage`. Приложение сериализует состояние в JSON и восстанавливает его при инициализации, что сохраняет данные пользователя между сессиями.
- **Оптимизация критического пути рендеринга:** Подключение скриптов вынесено в конец тега `body` и обернуто в слушатель события `DOMContentLoaded`. Это гарантирует быструю первичную отрисовку интерфейса (First Contentful Paint) до инициализации логики.
- **Валидация пользовательского ввода:** Реализована очистка строк с помощью метода `.trim()` для предотвращения создания пустых задач и забивания памяти некорректными данными.
- **Глобальная область видимости для динамических обработчиков:** Методы удаления и переключения статуса задач (`toggleTodo`, `deleteTodo`) явно привязаны к объекту `window`, что обеспечивает корректную работу инлайн-атрибутов `onchange` и `onclick`, генерируемых динамически через Template Literals.

## Стек 
- **HTML5:** Семантическая разметка: формы, списки, валидация полей ввода.
- **CSS3:** Flexbox-верстка, адаптивные интерфейсы, реализация состояний элементов через динамические классы (`.completed`).
- **JavaScript (ES6):** Стрелочные функции, деструктуризация, работа с массивами (`forEach`, `splice`), JSON-парсеры, асинхронные события DOM.

## Как запустить локально

1. Клонируйте репозиторий:
```bash
git clone https://github.com
```
2. Перейдите в папку с проектом:
```bash
cd ToDo_List
```
3. Откройте файл `index.html` в любом браузере (или запустите через расширение Live Server в VS Code).
